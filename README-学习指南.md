# Easy-Chat 学习指南

欢迎学习 Easy-Chat 项目！这是一个完整的即时通讯系统，基于 Go-Zero 微服务框架开发。

## 🎯 项目特点

- ✅ **完整的微服务架构**：User、Social、IM、Task 四大服务
- ✅ **详细的中文注释**：所有代码和配置文件都有详细的中文注释
- ✅ **完善的文档**：包含架构图、流程图、时序图等
- ✅ **真实的业务场景**：用户管理、好友管理、群组管理、即时通讯
- ✅ **现代化技术栈**：Go-Zero、gRPC、WebSocket、Kafka、Docker

## 🚀 快速开始

### 第一步：阅读架构文档

```bash
# 打开主文档
cat 项目架构与流程详解.md
```

建议阅读顺序：
1. 第1章：项目概述（理解项目是什么）
2. 第2章：整体架构（理解系统架构）
3. 第3章：技术栈详解（理解使用的技术）

### 第二步：查看基础设施

```bash
# 查看 Docker Compose 配置的中文注释
cat docker-compose.yaml.zh-CN
```

理解项目依赖的基础设施：
- Etcd（服务注册发现）
- Redis（缓存）
- MySQL（关系型数据库）
- MongoDB（文档数据库）
- Kafka（消息队列）
- APISIX（API 网关）

### 第三步：理解服务架构

阅读《项目架构与流程详解.md》第4章，理解四大服务：

1. **User 服务**：用户注册、登录、用户信息管理
2. **Social 服务**：好友管理、群组管理
3. **IM 服务**：即时通讯、消息发送、会话管理
4. **Task 服务**：异步任务处理、消息持久化

### 第四步：理解业务流程

阅读《项目架构与流程详解.md》第5-7章，通过流程图和时序图理解：

- 用户注册登录流程
- 添加好友流程
- 发送消息流程（单聊、群聊）
- 消息推送流程

### 第五步：查看代码实现

```bash
# 查看用户服务的 API 定义
cat apps/user/api/user.api.zh-CN

# 查看用户服务的 RPC 定义
cat apps/user/rpc/user.proto.zh-CN

# 查看用户服务的配置
cat apps/user/api/etc/dev/user.yaml.zh-CN
```

然后查看对应的 Go 代码，所有代码都有详细的中文注释。

### 第六步：部署运行

```bash
# 查看部署脚本的中文注释
cat deploy/script/release-test.sh.zh-CN

# 启动基础设施
docker-compose up -d

# 部署服务（查看脚本注释了解详细步骤）
cd deploy/script
./release-test.sh
```
